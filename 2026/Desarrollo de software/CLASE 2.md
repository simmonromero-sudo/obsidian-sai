### GIT

##### Ramas en git

- Permiten trabajar en paralelo sin afectar la ama principal.

Ideal para corregir errores, experimentar y desarrollar nuevas funcionalidades

**Merge**

2 tipos principales:

1. Fast-forward merge: Cunado no hay cambios divergentes. Simplemente "adelanta" la rama destino hacia nuestra rama
2. Three-away merge: Cuando ambas ramas tienen cambios diferentes. Git crea un nuevo commit que combina los cambios de ambas ramas 

**Conflicto en fusión de ramas**

Ocurren cuando:

- Se modifican las mismas líneas en diferentes ramas
- Git no puede determinar automáticamente que cambio conservar

<mark style="background:#9254de">Resolución manual:</mark>

1. Editar archivos para arreglar conflictos
2. Marcar como resueltos
3. Completar el merge

### GITHUB

#### Funcionalidades básicas

- Issues: Permiten hacer seguimiento de tareas, bugs o mejoras.
- Pull requests: son una forma estructurada de proponer cambios y revisarlos.
- Actions: Permiten automatizar flujos de trabajo como pruebas o deploys
- Projects: Facilita la gestión visual de tareas con tableros tipo Kanban
- Pages: Permite publicar sitios web directamente desde un repositorio
